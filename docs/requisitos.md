| ID   | Requisito Funcional                                                                 |
|------|--------------------------------------------------------------------------------------|
| RF01 | O sistema deve permitir ao usuário visualizar clínicas próximas com base em sua localização. |
| RF02 | O sistema deve permitir ao usuário selecionar uma clínica para atendimento.         |
| RF03 | O sistema deve permitir ao usuário selecionar a especialidade médica desejada.      |
| RF04 | O sistema deve permitir ao usuário visualizar os médicos disponíveis para a especialidade selecionada. |
| RF05 | O sistema deve permitir ao usuário selecionar um médico.                            |
| RF06 | O sistema deve exibir os dias disponíveis para atendimento do médico selecionado.   |
| RF07 | O sistema deve exibir os horários disponíveis para o dia selecionado.               |
| RF08 | O sistema deve permitir ao usuário selecionar data e horário para consulta.         |
| RF09 | O sistema deve solicitar os dados do usuário (nome, CPF e data de nascimento) para concluir o agendamento. |
| RF10 | O sistema deve validar o preenchimento dos dados obrigatórios antes de confirmar o agendamento. |
| RF11 | O sistema deve confirmar o agendamento após validação dos dados.                    |
| RF12 | O sistema deve permitir ao usuário iniciar o processo de agendamento de exames.     |
| RF13 | O sistema deve exigir o envio de um arquivo de encaminhamento médico para agendamento de exames (.png, .jpg, .pdf ou similares). |
| RF14 | O sistema deve validar o formato do arquivo enviado para exames.                    |
| RF15 | O sistema deve permitir prosseguir com o agendamento de exame após o envio válido do encaminhamento. |
| RF16 | O sistema deve seguir o mesmo fluxo de seleção (clínica, especialidade, médico, data e horário) para exames. |
| RF17 | O sistema deve permitir ao usuário visualizar os agendamentos realizados.           |
| RF18 | O sistema deve armazenar os dados dos agendamentos realizados.                      |
| RF19 | O sistema deve permitir o cancelamento de agendamentos.                             |
| RF20 | O sistema deve atualizar automaticamente a disponibilidade de horários após um agendamento ou cancelamento. |
| RF21 | O sistema deve enviar uma notificação ao usuário sobre agendamentos próximos com antecedência de 2 dias. |
| RF22 | O sistema deve permitir ao usuário cadastrar dependentes.                           |
| RF23 | O sistema deve permitir ao usuário editar os dados dos dependentes cadastrados.     |
| RF24 | O sistema deve permitir ao usuário remover dependentes cadastrados.                 |
| RF25 | O sistema deve permitir ao usuário selecionar um dependente para realizar agendamentos. |
| RF26 | O sistema deve associar agendamentos (consultas e exames) ao usuário ou dependente selecionado. |
| RF27 | O sistema deve exibir separadamente os agendamentos de cada dependente.              |
| RF28 | O sistema deve permitir ao usuário visualizar todos os agendamentos de um dependente selecionado. |
| RF29 | O sistema deve identificar claramente para quem (usuário ou dependente) cada agendamento pertence. |
